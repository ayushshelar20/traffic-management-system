📖 Step-by-Step Process

How It Works

Our AI controller follows a continuous loop — fetching data, analyzing congestion, making decisions, and adapting signals in real-time.

System Flowchart

Each cycle runs every few seconds, continuously optimizing traffic flow.

Step 1

📡 Fetch Traffic Data

The system collects real-time traffic data from APIs (Google Maps Distance Matrix) or uses simulated sensor data for each road approaching the intersection.

Step 2

🧠 Send to AI Controller

Raw travel time and delay data is fed into the AI decision engine for analysis. The controller maintains state for all four roads.

Step 3

📊 Analyze Congestion

Congestion ratio is calculated: congestion = current_travel_time / normal_travel_time. A ratio > 1.5 means heavy traffic.

Step 4

🏆 Identify Priority Road

The AI compares congestion levels across all roads and identifies which road has the highest traffic density and needs green signal most urgently.

Step 5

⏱️ Adjust Signal Timing

Green signal duration is dynamically calculated: higher congestion = longer green time (15–60 seconds). Other roads get proportional wait times.

Decision Point

🚑 Emergency Vehicle Check

The system checks if an emergency vehicle is detected on any road. This is the critical branching point in the control loop.

If YES

🚨 Emergency Override

Immediately switch the emergency road to GREEN and all others to RED. The override lasts until the vehicle passes, then normal control resumes.

If NO

✅ Continue Adaptive Control

Normal AI-based adaptive control continues. The system cycles through roads based on calculated priorities and dynamic timing.

Continuous

🔄 Repeat in Real-Time

The entire process repeats every cycle (configurable: 2–10 seconds), ensuring the intersection adapts continuously to changing conditions.

The Algorithm

Core logic behind the AI decision engine.

📐 Congestion Formula

congestion_ratio = current_time / normal_time

if ratio < 1.2 → Low traffic
if ratio 1.2–1.8 → Medium traffic
if ratio > 1.8 → High traffic

⏱️ Dynamic Timing

green_duration = base_time + (ratio × factor)

Minimum: 15 seconds
Maximum: 60 seconds
Emergency: Instant override
